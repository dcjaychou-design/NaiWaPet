# 架构说明

## 目标

NaiWaPet 的目标是在 Windows 11 x64 上提供无背景、动作完整、低依赖且可交互的桌面宠物，并保持构建与发布流程可复现。

## 运行时结构

```text
App / AppController
├── PetWindow                 透明 WPF 窗口和鼠标交互
├── SpriteAnimationPlayer    基于 Stopwatch 的 30 FPS 播放
│   ├── AnimationManifest    帧和图集位置
│   ├── AtlasCache           仅保留当前和相邻 PNG 图集
│   └── HitMask              当前帧像素点击检测
├── TrayIconService          托盘菜单与恢复入口
├── SettingsWindow           用户设置
├── NoticesWindow            版本、项目许可与运行时许可
├── SettingsStore            本地 JSON 原子写入
└── StartupManager           当前用户 Run 注册表项
```

## 动画方案

没有在运行时直接播放 MP4。普通视频播放器会带来矩形窗口、白底和额外解码依赖，也无法精确判断鼠标是否点到当前角色。构建阶段把视频转换为透明 PNG 图集和逐帧点击掩码；运行时只使用 WPF 自带的 PNG 解码器。

完整动画共 462 帧。每张图集为 8 × 8，播放器根据清单定位并裁剪单帧。缓存会在后台预读相邻图集的原始字节，但所有 WPF 图像对象只在界面线程解码和创建，既避免线程归属错误，也不会一次常驻全部高分辨率图像。

播放器进入新图集前会先清理不再相邻的已解码图集；完整素材冒烟测试也使用同一缓存边界，避免校验过程一次保留全部图集。动画清单和点击掩码在加载时会检查尺寸、帧数、连续性、源文件元数据及整数溢出。

音频是嵌入式 PCM WAV，默认随动画播放，也可以由用户随时关闭。

## 透明与点击

`AllowsTransparency=True` 创建逐像素透明窗口。窗口收到 `WM_NCHITTEST` 时，把屏幕坐标换算为动画帧坐标，再读取当前帧的位图掩码：透明像素返回 `HTTRANSPARENT`，鼠标会落到下方窗口；角色像素保持可点击。

## 交互与物理

拖动期间采集最近 140 毫秒的指针轨迹。释放速度足够高时进入简单重力模型，与当前显示器工作区边缘碰撞并衰减；低速时直接停下并保存位置。漫游使用短距离缓动和抛物线高度，不改变视频动画本身。

## 进程与持久化

命名 Mutex 保证单实例；第二次启动会在一个有限的短时间窗口内等待命名 EventWaitHandle，使正在初始化的已有实例也能收到显示请求。设置先写入带进程号和随机标识的临时文件，刷新到磁盘后再替换正式文件；损坏 JSON 会被保留为带时间戳的备份，然后恢复默认值。

开机启动使用当前用户的 Windows `Run` 注册表项。读取状态时会核对完整启动命令，移动程序后不会把指向旧路径的注册表值误报为已启用。

## 构建与校验边界

素材生成脚本从实际输入文件计算相对路径和 SHA-256，不接受项目目录以外的源视频。离线校验器会确认清单中的图集与已提交文件完全一致，并拒绝越出项目或素材目录的路径。

应用版本只在 `Directory.Build.props` 中定义。元数据检查器将其与程序集版本、标签、README、CHANGELOG、发布说明和文件名进行交叉检查。打包校验还会比较独立 EXE 与 ZIP 内 EXE、ZIP 文件清单、许可文本、Windows 文件版本和 SHA-256 文件。

正式发布拆分为无源码预检、只读 Windows 构建和最小写权限发布三个任务。第三方 Action 固定到完整提交 SHA；草稿中的全部文件重新下载并验证后才公开。不可变发布用于锁定公开后的标签和资产，并提供可验证的发布证明。

自包含 EXE 嵌入项目 GPL v3 许可、素材声明以及 .NET/WPF 10.0.10 的完整官方许可文本；便携 ZIP 同时以独立文件提供相同内容。许可查看窗口只读取嵌入式资源，不会联网。
