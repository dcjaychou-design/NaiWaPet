# 架构说明

## 目标

NaiWaPet 的目标是在 Windows 11 x64 上提供无背景、动作完整、低依赖且可交互的桌面宠物，并保持构建与发布流程可复现。

## 运行时结构

```text
App / AppController
├── PetWindow                 透明 WPF 窗口和鼠标交互
├── SpriteAnimationPlayer    基于 Stopwatch 的 30 FPS 播放
│   ├── AnimationManifest    帧和图集位置
│   ├── AtlasCache           仅保留当前和相邻 PNG 图集
│   └── HitMask              当前帧像素点击检测
├── TrayIconService          托盘菜单与恢复入口
├── SettingsWindow           用户设置
├── SettingsStore            本地 JSON 原子写入
└── StartupManager           当前用户 Run 注册表项
```

## 动画方案

没有在运行时直接播放 MP4。普通视频播放器会带来矩形窗口、白底和额外解码依赖，也无法精确判断鼠标是否点到当前角色。构建阶段把视频转换为透明 PNG 图集和逐帧点击掩码；运行时只使用 WPF 自带的 PNG 解码器。

完整动画共 462 帧。每张图集为 8 × 8，播放器根据清单定位并裁剪单帧。缓存会在后台预读相邻图集的原始字节，但所有 WPF 图像对象只在界面线程解码和创建，既避免线程归属错误，也不会一次常驻全部高分辨率图像。

音频是嵌入式 PCM WAV，默认随动画播放，也可以由用户随时关闭。

## 透明与点击

`AllowsTransparency=True` 创建逐像素透明窗口。窗口收到 `WM_NCHITTEST` 时，把屏幕坐标换算为动画帧坐标，再读取当前帧的位图掩码：透明像素返回 `HTTRANSPARENT`，鼠标会落到下方窗口；角色像素保持可点击。

## 交互与物理

拖动期间采集最近 140 毫秒的指针轨迹。释放速度足够高时进入简单重力模型，与当前显示器工作区边缘碰撞并衰减；低速时直接停下并保存位置。漫游使用短距离缓动和抛物线高度，不改变视频动画本身。

## 进程与持久化

命名 Mutex 保证单实例；第二次启动会触发命名 EventWaitHandle，使已有实例显示并播放动作。设置按原子替换方式保存，损坏 JSON 会被保留为带时间戳的备份后恢复默认值。
